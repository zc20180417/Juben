# Genre Profile: revenge_palace

本 profile 只定义“复仇 / 宫斗 / 上位”题材的推进方式。
它建立在 `harness/framework/write-contract.md` 与 `harness/framework/writer-style.md` 之上。

## 题材引擎
- 观众主要追看三件事：
  - 仇怎么报
  - 人怎么上位
  - 局怎么反转
- 主角的推进方式通常是：藏锋 → 试探 → 借刀 → 反杀 → 上位
- 情绪回报优先级：羞辱反击 > 地位逆转 > 权力位移 > 旧仇回收

## 冲突推进
- 冲突优先外化为：礼法压迫、身份羞辱、资源争夺、圣心争夺、权位移动
- 反派不能只会嘴上压人；必须能在规则、位置、资源上造成真实威胁
- 主角不能全靠“我早就算到了”推进；要有代价、受挫、误判或被迫提前出手的时刻

## 受挫与翻盘
- 受挫不是简单吃亏，而是：
  - 计划被打断
  - 盟友受损
  - 身份暴露风险上升
  - 情感牵连导致判断波动
- 翻盘要靠前面埋下的筹码，而不是空降聪明
- 复仇类主角可以强，但不能全程无破绽

## 场次倾向
- 常见结构：
  - 铺垫 → 试探 / 执行 → 结果 / 余波
  - 羞辱 → 忍让 → 借势反击
  - 平静铺局 → 突袭揭底
- 允许一场只做“余波”或“安静蓄压”，用来给高潮拉开落差

## 记忆唤醒
- 复仇得手、身份揭露、旧仇回收时，优先考虑记忆唤醒
- 可用手段：
  - 旧物
  - 旧台词
  - 旧伤
  - 闪回
  - 视觉平行
- 记忆唤醒要短、硬、具体，不能写成“她想起了当年”

## 禁忌写法
- 不要把主角写成单纯嘴狠的“神机军师”
- 不要让每一场都只靠 `os` 宣布“上钩了”
- 不要让反派只有功能，没有位置和手段
- 不要把宫斗写成纯台词斗嘴，必须让规则、地位、名分产生实际后果
