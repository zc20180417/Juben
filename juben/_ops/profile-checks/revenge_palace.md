# Profile Check: revenge_palace

本文件只检查 `revenge_palace` 题材与 `cn_paid_microdrama` 分发假设下的特有要求。
不覆盖 `runtime-core.md` 与 base aligner 已经负责的跨题材规则。

## 题材检查
- 第1-10集：每2-3集应出现一次明显的小高潮、打脸或关系反转
- 第11-20集：免费段末尾应持续抬高风险，不能一路平推
- 后段：重点从单次打脸转向局势推进、位置变化与权力位移

## 主角受挫 / 翻盘
- 连续 5 集主角计划均无受挫：WARNING
- 连续 8 集主角计划均无受挫：FAIL
- 翻盘若没有前文筹码支撑，判逻辑不足

## 复仇类记忆唤醒
- 复仇得手 / 仇人落败 / 身份曝光 / 旧仇回收时，应优先检查是否使用了具体记忆唤醒手段
- 若题材上明显该有“今昔对照”却完全没有，判 FAIL

## 平台分发假设
- `distribution_mode: cn_paid_microdrama` 下，18-21 集应出现付费转化节点
- 18-20 集至少要有一次情感高潮或重大压迫升级
- 20 集结尾或 21 集开头应有显著悬念 / 转折
- 付费后首集应给即时情感回报，不能只延迟兑现

## 输出要求
- 检查结果只补充题材与平台特有结论
- 与 base aligner 冲突时，先确认是否属于 profile 特有要求，再决定是否覆盖
