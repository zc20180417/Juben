# 微短剧剧本创作 Agent 系统

基于 AI Agent 架构的中国微短剧剧本全流程创作系统。通过多 Agent 协作，完成从创意到成品剧本的创作。

## 系统架构

```
用户指令
  │
  ▼
主Agent（编剧/导演）──── 统筹创作流程，负责实际写作
  │
  ├── script-aligner ─── 校验剧情逻辑、人物一致性、伏笔管理
  └── script-recorder ── 记录创作进度、版本快照、质量统计
```

## 创作流程

```
/outline  故事大纲  ──┐
/character 人物小传  ──┤  每步均走完整调用链：
/catalog  集目录    ──┤  创作 → aligner检查 → 写入 → recorder记录
/write N  第N集正文 ──┘
```

## 文件结构

```
project/
├── CLAUDE.md                 # 主Agent配置和项目规则
├── script-aligner.md         # 校验Agent配置
├── script-recorder.md        # 记录Agent配置
├── short-drama-cn.md         # 短剧创作法则参考
├── script.progress.md        # 创作进度记录（自动维护）
├── outline.md                # 故事大纲（当前版本）
├── character.md              # 人物小传（当前版本）
├── episode_index.md          # 集目录（当前版本）
├── versions/                 # 版本快照目录（自动维护）
│   ├── v001_outline.md
│   ├── v001_character.md
│   └── changelog.md          # 变更日志
└── episodes/
    ├── EP-01.md
    ├── EP-02.md
    └── ...
```

## 优化记录

### v2.0 优化方案（2026-03-30）

针对 v1.0 系统的六个核心缺点进行改造。

---

#### 一、Sub-agent 调度：关键词触发 → 流程节点强制

**问题**：v1.0 依赖 LLM 识别用户自然语言中的关键词（如"推翻/改名/改设定"）来触发 sub-agent，容易漏触发或误触发。

**改造**：取消所有基于关键词的模糊触发，改为流程节点绑定。

| 调用点 | 触发时机 | 硬性规则 |
|-------|---------|---------|
| script-aligner | 每次写入 outline/character/episode_index/EP-XX **之前** | 无 PASS 不允许写入 |
| script-recorder | 每次成功写入上述文件 **之后** | 写入即记录，无需判断意图 |

用户提出任何修改 → 主Agent修改 → 走同一条完整调用链，不再区分"设定调整语""约束确立语"等类别。

**涉及文件**：`CLAUDE.md` [自动触发规则]

---

#### 二、Aligner 检查标准量化 + 对抗性检查

**问题**：v1.0 的检查标准是描述性的（如"对话是否简短有力"），LLM 自己写自己查容易自我放过。

**改造**：增加三层检查机制。

**量化检查清单**（硬性可计数指标，不满足即 FAIL）：

| 维度 | 指标 |
|-----|------|
| 单集 | 场景数 ≤ 3、单句对话 ≤ 20字、连续对话 ≤ 6轮、每集 ≥ 1个冲突/反转、集尾必须有钩子 |
| 跨集 | 连续压抑 ≤ 3集、每5集 ≥ 1个爽点、伏笔超20集未回收 WARNING |
| 付费节点 | 第18-20集 ≥ 1个情感高潮、第20-21集必须有重大悬念 |

**对抗性检查**（防止自我认同偏差）：

| 测试 | 方法 | FAIL条件 |
|-----|------|---------|
| 角色替换 | 将角色A对话换成角色B | 读起来毫无违和感 |
| 删除测试 | 假设删掉某个场景 | 剧情完全不受影响 |
| 逻辑反推 | 从结果反推原因 | 找不到因果链条 |
| 钩子有效性 | 读完集尾最后一段 | 没产生"接下来会怎样"的疑问 |

**级联影响检查**（outline/character 修改时触发）：
- 扫描已完成的 EP，找出与修改内容冲突的集数
- 输出影响报告和建议处理顺序
- 不影响当前内容的 PASS/FAIL 判定

**涉及文件**：`script-aligner.md` [量化检查清单] [对抗性检查] [级联影响检查]，检查流程从4步扩展为7步

---

#### 三、版本管理与级联更新

**问题**：v1.0 没有版本追踪，改了大纲后无法评估对已写内容的影响。

**改造**：在 script-recorder 中增加版本快照功能。

**快照流程**：
```
outline.md 或 character.md 被修改
  → 将修改前版本复制到 versions/v{序号}_{文件名}.md
  → 在 versions/changelog.md 追加变更记录
  → 更新 script.progress.md 版本记录区块
```

**changelog.md 格式**：
```markdown
## v002 - 2026-03-30 14:30
- 文件：outline.md
- 变更：将男主身份从"退伍军人"改为"隐退特种兵"
- 原因：用户认为不够有爽感
- 影响范围：EP-01、EP-05、EP-12
```

配合 aligner 的级联影响检查，修改后可以明确知道哪些已完成的集需要联动修改。

**涉及文件**：`CLAUDE.md` [文件结构]、`script-recorder.md` [版本快照] [版本快照规则]

---

#### 四、上下文分层加载策略

**问题**：v1.0 写每集时都要读取全部前置文档，项目变大后可能超出上下文窗口。

**改造**：按需分层加载。

| 层级 | 内容 | 加载策略 |
|-----|------|---------|
| 必读 | episode_index.md、character.md、伏笔总表、高频问题 TOP3 | 每集都读 |
| 按需读 | outline.md 对应段落、前2集正文或前1集衔接摘要、相关伏笔埋设集 | 与当前集相关时读 |
| 不读 | 无关的早期EP正文、versions/ 历史版本、changelog | 跳过 |

同时在 script.progress.md 的分集记录中增加**前情衔接摘要**字段，2-3句话描述本集结尾状态，写下一集时读摘要即可，不用读全文。

**涉及文件**：`CLAUDE.md` [剧本正文创作阶段]、`script-recorder.md` 模板和分集合并处理

---

#### 五、场景级用户确认

**问题**：v1.0 整集写完再让用户看，颗粒度太粗，返工成本高。

**改造**：在正文创作中增加"先出结构、再写正文"的两步确认。

```
/write N
  → 分层加载上下文
  → 输出场景骨架（每场的目的、关键事件、钩子）
  → 等待用户确认或调整
  → 确认后创作正文
  → 走 aligner → 写入 → recorder 流程
```

场景骨架包含：场景标题、日/夜内/外、地点、本场目的、关键事件、结尾钩子、本集爽点和伏笔规划。

**涉及文件**：`CLAUDE.md` [剧本正文创作阶段] 第二步

---

#### 六、质量反馈闭环

**问题**：v1.0 有检查但没有统计和趋势分析，无法持续改进。

**改造**：在 script.progress.md 中增加质量统计区块。

**检查记录表**：每集的检查次数、首次结果、主要问题类型

**高频问题 TOP3**：每次更新时重新统计，按频次排序，附带应对建议
```
1. 对话过长（出现5次）→ 写对话时先控制在15字内
2. 场景数超标（出现3次）→ 规划阶段就限制为1-3场
```

**质量趋势**：每10集统计平均检查次数、最稳定/薄弱方面

主Agent创作新集时读取"高频问题 TOP3"作为重点注意事项，形成改进闭环。

**涉及文件**：`script-recorder.md` 模板 [质量统计]、分集记录 [aligner检查记录]、全局信息更新

---

### 改造优先级

| 优先级 | 改造项 | 原因 |
|-------|-------|------|
| P0 | 调度节点化 | 根基问题，不改则其他优化都不可靠 |
| P0 | 检查量化 | 直接决定输出质量 |
| P1 | 场景级确认 | 改善用户体验，减少返工 |
| P1 | 上下文分层 | 项目变大后的必要条件 |
| P2 | 版本管理 | 中后期才会频繁遇到 |
| P2 | 质量统计 | 锦上添花，但长期价值高 |

### v2.1 内容丰满度优化（2026-03-31）

v2.0 上线后用小说《墨凰谋：庶女上位录》实测（Codex + OPENAI.md），发现生成的剧本内容过于简陋：每集仅 33-47 行，△描写只有1句话，缺失内心独白、表情特写、闪回、镜头语言、音效提示。根因是 EP 模板太骨架化 + 创作法则只有上限没有下限。

---

#### 七、EP 模板丰富化：从骨架升级为完整剧本格式

**问题**：v2.0 的 EP 模板只定义了 △ + 对话，没有为旁白、镜头、音效、闪回留位置，AI 不知道要写这些内容。

**改造**：重写 EP 模板，新增6种标记元素。

| 标记 | 用途 | 要求 |
|-----|------|------|
| `△` | 场景/动作描写 | 每条 2-4 句，要有画面感 |
| `os` | 内心独白 | 每集 ≥ 1 处，权谋类 ≥ 2 处 |
| `vo` | 画外音/旁白 | 回忆、转场、点题时使用 |
| `闪回` | 回忆画面 | 复仇/伏笔回收/身份曝光必须配 |
| `♪` | 音效/配乐 | 每场 ≥ 1 处 |
| `【镜头】` | 镜头语言 | 每集 ≥ 1 处 |

模板包含完整示例片段，展示各元素如何配合使用。

**涉及文件**：`CLAUDE.md` [episodes模板]、`OPENAI.md` EP-XX.md 模板

---

#### 八、内容丰满度硬性下限

**问题**：v2.0 的量化检查只有上限（场景 ≤ 3个、对话 ≤ 20字），没有下限，AI 默认"越精简越好"导致内容严重不足。

**改造**：在创作法则和 aligner 检查中同时增加下限指标。

| 指标 | 下限 | 之前实际值 |
|------|------|-----------|
| 单集总行数 | ≥ 70行（关键集 ≥ 90行） | 33-47行 |
| 单场景行数 | ≥ 25行 | 15-22行 |
| △描写单条 | ≥ 2句 | 1句 |
| 每场对话轮数 | ≥ 6轮 | 3-5轮 |
| os/vo | 每集 ≥ 1处 | 0处 |
| 表情特写 | 每集 ≥ 2处 | 0处 |
| 连续纯对话上限 | ≤ 4句 | 无限制 |

同时增加**"简短有力"正确理解**段落，明确纠正"精炼 = 少写"的误解。

**涉及文件**：`CLAUDE.md` [内容丰满度要求]、`OPENAI.md` 内容丰满度要求、`short-drama-cn.md` 丰满度规则、`script-aligner.md` [量化检查清单]

---

#### 九、对抗性检查扩展：4项 → 6项

**问题**：原有4项对抗性检查不涉及内容丰满度，无法拦截"通过了逻辑检查但内容简陋"的情况。

**改造**：新增2项对抗性检查。

| 新增测试 | 方法 | FAIL条件 |
|---------|------|---------|
| 画面感测试 | 抽取任意△描写，闭眼能否想象出具体画面 | 无法想象光线/人物姿态/环境细节 |
| 表里不一测试 | 检查os是否与表面言行形成反差 | os只是重复对话意思，无反差 |

**涉及文件**：`script-aligner.md` [对抗性检查]、`OPENAI.md` 对抗性检查

---

#### 十、闪回专项指标

**问题**：复仇剧的核心爽感来自"回忆当年→现在报仇"的对比，但生成内容中零闪回。

**改造**：在 aligner 中增加闪回专项检查。

以下场景类型必须包含闪回，缺失即 FAIL：
- 复仇得手/仇人落败时 → 闪回当年受害画面
- 伏笔回收时 → 闪回伏笔埋设画面
- 身份曝光/真相揭露时 → 闪回相关暗示细节

闪回内容 1-3 句，必须具体到画面细节。

**涉及文件**：`script-aligner.md` [EP-XX 闪回专项指标]

### v2.2 收敛优化（2026-03-31）

v2.1 实测反馈后的收敛调整，核心原则：**规则应该约束"效果"，而不是约束"手段"。**

---

#### 收敛一：行数从硬门槛改为参考线 + 结构性门槛

**问题**：v2.1 把"单集 ≥ 70行"作为 FAIL 条件，会导致 AI 凑行数而非加内容——把每条△拉长到4句，对话间塞无意义的动作描写，反而破坏节奏。

**改造**：

| 改造前 | 改造后 |
|-------|-------|
| 总行数 ≥ 70行 = 硬性 FAIL | 总行数建议 ≥ 70行 = 参考预判线 |
| 单场 ≥ 25行 = 硬性 FAIL | 单场建议 ≥ 25行 = 参考预判线 |

真正判 PASS/FAIL 的条件改为结构性要素：
- △描写每条 ≥ 2句
- 每场对话 ≥ 6轮交锋
- 连续纯对话 ≤ 4句
- os ≥ 1处（权谋类 ≥ 2处）
- 表情特写 ≥ 2处
- 冲突/反转 ≥ 1个
- 集尾钩子
- 记忆唤醒手段（见下）

低于参考行数时 aligner 会重点审查结构性门槛；高于参考行数但结构性门槛不达标仍然 FAIL。

**涉及文件**：`CLAUDE.md` [内容丰满度要求]、`OPENAI.md` 内容丰满度要求 + 校验标准、`script-aligner.md` [量化检查清单]、`short-drama-cn.md` 丰满度规则

---

#### 收敛二：闪回一刀切改为记忆唤醒手段体系

**问题**：v2.1 要求复仇/伏笔回收/身份曝光场景"必须有闪回"，但闪回只是唤醒观众记忆的手段之一。强制使用闪回会错过更贴合角色气质的写法——比如角色触碰旧物、引用旧台词，可能比直接闪回更含蓄、更有层次。

**改造**：将"必须有闪回"改为"必须有记忆唤醒手段"，列举六种等效手段：

| 手段 | 说明 | 适用场景举例 |
|------|------|-------------|
| 闪回 | 直接插入过去画面 | 重大转折、终极复仇 |
| 旧物触发 | 触碰/看到过去相关物件 | 含蓄角色、情感克制场景 |
| 旧台词复现 | 引用过去某人说过的话 | os中形成今昔对比 |
| 旧伤/旧痕 | 露出伤疤、摩挲旧痕 | 身体记忆唤起的场景 |
| 证据对照 | 新证据与过去细节并置 | 揭秘/真相大白 |
| 视觉平行 | 同一地点/姿态，不同时间 | 地位反转、命运轮回 |

选择哪种手段由编剧根据角色性格和场景情绪决定。不合格的写法（"她想起了往事""回忆涌上心头"）仍然 FAIL——必须写出具体的物件/台词/画面。

**涉及文件**：`CLAUDE.md` [记忆唤醒手段] + EP模板、`OPENAI.md` 记忆唤醒手段 + EP模板 + 校验标准、`script-aligner.md` [EP-XX 记忆唤醒指标]、`short-drama-cn.md` 丰满度规则

### v2.3 整季统筹优化（2026-03-31）

v2.2 解决了"单集太薄"的问题，但在《墨凰谋：庶女上位录》整季实测中暴露出新的结构性缺陷：前期质量明显高于后期，人物说话方式逐渐同质化，后半段更像"事件推进"而不是"棋局推进"。根因不是单条规则失效，而是系统仍停留在**单集思维**，缺少整季统筹机制。

---

#### 十一、质量基准锚：防止后期自然滑坡

**问题**：AI 连续写到第30集以后，已经不会主动参考 EP01-EP03 的质感，只会延续最近几集的惯性。如果第20集后微微变薄，后面会一路变薄，但单集 aligner 仍可能 PASS。

**改造**：
- 新增 `quality.anchor.md`
- 用前3集或用户认可样集抽取质量锚：
  - 场景厚度
  - 对话节奏
  - os 的用途
  - 表情/镜头/音效密度
  - 代表性爽点打法
- 每完成5集，必须先与质量锚对标；若最近一批明显弱于锚点，不得继续往后写

**涉及文件**：`CLAUDE.md` [整季质量锚机制]、`script-aligner.md` [整季质量锚指标]、`OPENAI.md` 整季质量锚机制、`script-recorder.md` 质量锚规则

---

#### 十二、批次生成 + 状态快照：把上下文控制在友好区间

**问题**：一次性跨越几十集正文创作，注意力必然被摊薄。即使规则很严，长上下文也会让模型越来越依赖最近几集的"惯性质量"。

**改造**：
- 默认按 **5集一批** 推进，单批最多不超过10集
- 每批开始前，先生成本批小弧光目标
- 每批结束后，强制更新：
  - `story.state.md`
  - `relationship.board.md`
  - `open_loops.md`
  - `script.progress.md`
- 下一批只读高价值状态文件 + 最近1-2集正文，而不是回读整季全文

这实际上把系统从"长上下文硬顶"改成"滚动上下文 + 状态压缩"。

**涉及文件**：`CLAUDE.md` [批量生成模式]、`script-recorder.md` 批次状态快照规则、`OPENAI.md` 批量生成模式

---

#### 十三、角色语音指纹：解决角色同质化

**问题**：v2.2 的 `character.md` 只定义了人物功能，没有定义"她怎么说话"。结果越往后写，沈青鸾、皇后、沈夫人、萧景珩都可能滑向同一种"短、冷、狠"腔。

**改造**：
- 在 `character.md` 模板中加入 **语音指纹** 字段：
  - 语速
  - 句式
  - 口头习惯
  - 禁用表达
  - 狠的方式
- aligner 增加 **角色语音指纹检查**

这样角色不仅有"功能"，还有稳定的"声纹"。

**涉及文件**：`CLAUDE.md` [character.md模板]、`script-aligner.md` [角色语音指纹指标]、`OPENAI.md` `character.md` 模板、`short-drama-cn.md` 角色声纹规则

---

#### 十四、阶段节奏标签：前半段宫斗，后半段权谋

**问题**：以前所有阶段共用同一套单集逻辑，导致后半段依然按前半段"打脸/钩子/反转"思路在写，结果事件是对的，局势却不够厚。

**改造**：
- 新增 **阶段节奏标签**：
  - 引流/宫斗期
  - 猎杀/上位期
  - 权谋/决战期
- 不同阶段使用不同优先级：
  - 前段重羞辱、反击、得宠、立威
  - 中段重反派梯度清算、位份和关系位移
  - 后段重储位、兵权、遗诏、废帝、天下格局变化
- aligner 增加 **阶段节奏检查**

**涉及文件**：`CLAUDE.md` [阶段节奏标签]、`script-aligner.md` [阶段节奏指标]、`OPENAI.md` 阶段节奏标签、`short-drama-cn.md` 后半程权谋模式

---

#### 十五、留白层级 + 标记密度曲线：解决显性化和节奏扁平

**问题**：
- 规则过度鼓励"说清楚"，导致人物容易把狠和算计直接说成台词或os
- 同时又要求每场都塞满标记，造成低谷场和高潮场密度差不多，节奏被压平

**改造**：
- 新增 **留白层级**：
  - A类戏：必须说清
  - B类戏：说半层，留半层给观众
  - C类戏：优先用动作、停顿、眼神、镜头传达
- 新增 **节奏密度曲线**：
  - 低谷场可以极简
  - 常规场保持完整基本盘
  - 高潮场标记密度显著增强
- aligner 增加：
  - 显性化/留白检查
  - 标记密度曲线检查

**涉及文件**：`CLAUDE.md` [留白层级] [节奏密度曲线]、`script-aligner.md` 显性化/留白指标 + 标记密度曲线指标、`OPENAI.md` 对应章节、`short-drama-cn.md` 留白层级规则 + 节奏密度曲线

---

#### 十六、可选 sub-agent 执行模式

**问题**：整季生成时，单个主Agent同时承担批次规划、单集写作、连续性校验、状态记录四种任务，容易导致注意力分散。

**改造**：
- 若运行环境支持 sub-agent 或子任务，可逻辑拆分为：
  - `batch-planner`
  - `episode-writer`
  - `continuity-aligner`
  - `state-recorder`
- 若运行环境不支持，则主Agent必须按同样顺序自行模拟

这不是强依赖，但为未来的 agent-native 工作流预留了接口。

**涉及文件**：`CLAUDE.md` [总体规则] [批量生成模式]、`OPENAI.md` 批量生成模式

### v2.3 结果

v2.3 的核心不是让单集更华丽，而是让系统从：

**"会写一集短剧"**

升级为：

**"能控制一整季短剧的质量、节奏和角色一致性"**

重点解决：
- 后期变薄
- 过度显性化
- 角色同质化
- 后段权谋乏力
- 标记密度扁平

#### 修复二：理清批量生成模式与场景确认的优先级

**问题**：`/write` 流程的第二步要求"逐集场景预览等待用户确认"，但批量生成模式要求"自动连续生成5集"。两个逻辑并存但没有说清谁优先，导致批量模式实际无法自动推进。

**修复**：在批量生成模式中明确三种确认策略：
- 默认：每批开始时一次性输出本批所有集的场景骨架，用户确认整批后逐集写正文
- 全自动：用户明确授权后可跳过场景预览，每批结束后展示摘要供审阅
- 单集模式不受影响，仍然逐集确认

同时在第二步加注说明，指向批量模式的确认规则。

**涉及文件**：`CLAUDE.md` [批量生成模式] + 第二步、`OPENAI.md` 批量生成模式 + 第二步
