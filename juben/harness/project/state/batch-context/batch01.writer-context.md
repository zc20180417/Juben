# 写作合同（Write Contract）
<!-- PROMPT-REFERENCE: Writer prompt may reference stable contract sections below. Keep section ids stable when editing. -->

本文件定义 writer 的通用写作边界，不包含项目专有章节映射。
具体写法风格补充见 `harness/framework/writer-style.md`。
本文件优先定义硬门槛、可执行边界和尽量可验证的规则；若与 `writer-style.md` 冲突，以本文件为准。

## 写作姿态（Writing Posture）
<!-- SECTION:WRITING_POSTURE -->
- `△` 只写摄影机能拍到的内容
- `os` 是主观窗口，但不能承担剧情讲解职责
- 完整气口优先于碎句节拍（见下方“对白排版（Dialogue Formatting）”）
- 具体场景打法、叙事姿态与风格警戒统一遵循 `writer-style.md`

## 标记格式（Marker Formatting）
<!-- SECTION:MARKER_FORMAT -->
- `△`、`♪`、`【镜头】：`、`角色：`、`角色（os）：` 各自独占一行，不与其他标记混排
- `△` 只承担可见动作、环境、道具、姿态和物理细节
- `♪` 只承担环境声、音效、节奏停顿、配乐切入或刻意沉默
- `角色（os）：` 必须带明确角色名，不写成匿名念头
- `♪` 与 `【镜头】：` 是壳层语法，不是装饰；只有在新增节奏、视角或信息落点时才写，不能机械铺满

## 对白规则（Dialogue Rules）
<!-- SECTION:DIALOGUE_RULES -->
- 允许换说法，不允许换态度
- 不得把受压、求生、陈述改成挑衅、机锋、评判或聪明压人
- 保留回应句时，必须保留上句触发语义
- 不得把任何角色统一压成碎句机器（包括主角）
- 原著对应段若以施救、守夜、观察、潜伏、追踪为主，禁止为撑节奏主动补密集对白、机锋、狠话或关系宣告句
- 原著若只发生一次短试探或短问答，不得擅自扩成多轮连续盘问；能在一场里完成的，不扩成“户籍式”追问链
- 一句台词若只是在重复镜头里已成立的动作判断，且删掉后场意不变，应删
- 角色尚未走到那一步时，不得替他提前说出关系、亏欠、信任或心软的结论句

## 对白排版（Dialogue Formatting）
<!-- SECTION:DIALOGUE_FORMATTING -->
"短句"指角色说话内容简洁，不是剧本排版上每行都要断开。区分方式：

- **一个语义单元 = 一行**。角色一口气说完的话，即使由几个短句组成，也写在同一行
- 只有在语义上有**停顿、转折、施压**时才换行——换行本身就是节奏工具
- 错误示范（机械断行）：
  ```
  萧景珩：让朕的嫔妃当众献舞。
  萧景珩：你当她是歌姬？
  ```
- 正确示范（语义换行）：
  ```
  萧景珩：让朕的嫔妃当众献舞，你当她是歌姬？
  ```
- 当角色**刻意施压、逼问、一句一句钉**时，才拆成多行：
  ```
  萧景珩：聚聚。
  萧景珩：聚到她掉眼泪？
  ```
  ↑ 这里"聚聚"是冷重复，第二句是反问施压——两句之间有刻意停顿，所以分行。
- 判断标准：如果拆开后两行之间**读不出停顿感**，就不该拆

## `OS` 规则（OS Rules）
<!-- SECTION:OS_RULES -->
- `os` 必须一眼读清主语、对象和判断落点
- `os` 不得悬空到需要读者自己补半句
- `os` 不解释剧情，不提前宣布计划得手，不把观众判断说穿
- `os` 应优先承担：表里不一、代价感、被触发的旧记忆
- 删掉后若场意、人物动机和关系变化都仍然成立，该 `os` 应删
- `os` 不得把角色尚未承认的关系判断、情绪结论或观众本该自己感受到的变化直接说透

## 场次规则（Scene Rules）
<!-- SECTION:SCENE_RULES -->
- 每集至少 2 场；场次数由功能完成决定，不设置硬上限，但不得用空场凑时长
- 每集必须有明确戏剧推进；“推进”至少落在其一：关系位移、新信息、目标变化、新威胁、决定、代价
- 一场只主打一项功能；若同一场同时承担两个强功能，应拆场而不是糊成总结场
- 集尾要么有前推力，要么有强闭环落点
- 即时动作钩子只有在相邻下一集第一场会兑现时才允许使用
- 相邻两集连续承接同一时间地点时，下一集必须尽快进入新增推进，不得把上一集尾部已完成动作整场重演
- 低对白场允许主要由动作、停顿、视线、道具和位置变化承担推进，不强制补对白填满
- 场景开头的环境交代只保留最有效的 1-2 个视觉 / 听觉符号；若删掉一半晨光、残滴、白气、风声后信息不变，说明写密了
- EP-01 到 EP-03 执行黄金开场：第一场第一组可拍动作或第一句台词必须直接进入异常、冲突、羞辱、逼选、误认、证据或身份错位，不得慢铺人物小传
- EP-01 必须在本集内完成“异常入局 + 第一轮关系冲突 + 强钩”；不得只写事件前夜或停在风暴将起

## 改编姿态（Adaptation Posture）
<!-- SECTION:ADAPTATION_POSTURE -->
- 改编不是压缩字数，而是重分配信息载体
- 当前目标总时长以 `run.manifest.md` 的 `target_total_minutes` 为准；应按该总时长对应的高密度完整剧集理解，不按 80-100 集付费长链条灌水
- 剧中人物姓名必须与原著姓名完全不同；不得沿用原名，不得只改姓或只改名，不得用同音/近音/叠字变体规避
- 改名只改变称呼，不改变人物功能、关系、弧光和 source 顺序
- 若 batch brief、source.map、character.md 或 voice-anchor.md 仍出现原著姓名，writer 应按已给出的改名映射统一替换成剧名；没有映射时必须先在草稿开头外的内部判断中确定映射，但不得把映射说明写进剧本文本
- 先保主线推进，再保关键关系变化，再保爆点与反转，最后保世界规则与主题表达
- 原著叙述信息优先外化成：动作、停顿、道具、环境变化、对话反应
- 第一人称原著默认改写成第三人称可拍视角，不直接保留“我”叙述腔
- 角色心理先尝试外化成表情、身体动作、选择和说话方式；只有画面承载不了时才进入 `os`
- 原著里”写出来很好看”的段落，不等于搬成剧本就有效
- 剧本语言只保留人物会说的话、摄影机能拍到的事、剪辑能组织出的信息
- 原著叙述腔、优美句子、哲理句、旁白总结句不直接入正文
- `△` / `vo` / `os` 不直接沿用小说式“我如何看见、我如何觉得、我如何走进去”的叙述句
- 当原著气口与“短剧补词”冲突时，优先保原著气口；宁留白，不补模板狠话
- 原著低对白源段，不得硬改成高交锋、高机锋或高表态场
- 原著若把压力主要放在某一方“被看、被试、怕暴露”的感受上，改编时优先保护这条单侧紧张线，不急着写成双方势均力敌的对手戏
- 身份、姓氏、名号若在原著里天然带分量，剧本里要用可感的联想或后果把重量落地，不能只写成抽象氛围

## 声纹锚点（Voice Anchor）
<!-- SECTION:VOICE_ANCHOR_USAGE -->
- writer 优先读取 `voice-anchor.md`
- 若 `voice-anchor.md` 不存在、仅剩模板占位或未列出当前角色，回退到 `character.md` 中的语音指纹
- 只继承说话方式，不搬运小说叙述腔
- 角色台词必须符合声纹锚中的句长基准、句式基准和禁用表达

## 成稿风格（Writer Style）
<!-- SECTION:STYLE_HANDOFF -->
- 叙事立场、场景打法、对话打法、集级节奏和风格警戒统一定义在 `writer-style.md`
- contract 负责门槛，`writer-style.md` 负责稳定成稿时的具体执行方式
- 两者冲突时，以本文件的边界与硬门槛为准

## 原著保真规则（Original-Fidelity Rules）
<!-- SECTION:ORIGINAL_FIDELITY -->
- 章节顺序优先
- 人名不属于本项目的保真对象；保真的是关系、功能、事件顺序和情绪逻辑
- 一章可拆多集，多个相邻章节可合拍
- 不得抢跑后续章节大事件
- 原著自然发生的关键节点，不得无依据改成预谋布局
- 原著概述性信息，不得擅自扩成新的人名、具体事故或完整事件链
- 若原著某场核心是”某个人第一次单独注意到她 / 记住她 / 被她刺中”，不得扩成多人围观、多人评价的群像场
- 这类单点被看见、单点被记住、单点被刺中的关系场，优先保护其专属性
- 原著若只给一轮简短试探，优先保“短”和“准”；不得为补时长把它改写成多场连续问身世、问来处、问家门的流程场

## 交稿前自检（Pre-Submit Check）
<!-- SECTION:PRE_SUBMIT_CHECK -->
- 检查成稿中是否出现原著人物姓名；如出现，必须替换成剧名
- 若当前集是 EP-01 到 EP-03，检查开场前 10-20 秒是否已经进入强冲突或强悬念；若还在铺环境、介绍人物或等待事件发生，必须重写开场
- 检查本集是否有主戏闭环：进入、对抗/变化、结果或钩子；若只是几个事件点顺排，必须重组
- 当前集若改编自低对白源段，检查是否被写成“靠台词撑戏”；若是，先删台词，再补动作
- 检查是否出现只为立人设而写的冷句、狠话、机锋句；若删掉后人物仍成立，则删
- 检查 `os` 是否重复了画面、台词或观众已能感受到的信息；若重复，则删
- 检查试探场是否被扩成多轮盘问；若删去一半问答后主信息仍成立，说明写多了，应压短
- 检查压力是否被错误写成“双方都很会试探”的均衡对戏；若原著核心其实是谁更怕暴露，就把镜头和感受重新压回那个人
- 检查开场环境是否在堆同层氛围词；若雨声、灯影、白气、晨光删去一半后场意不变，应删
- 检查 `♪` / `【镜头】` 是否只是重复已成立画面；若不承担新增信息或节奏转折，应删
- 检查集尾是否有新增推进，而不是仅停在情绪余波或平地对视

## 角色知识边界（Character Knowledge Boundary）
<!-- SECTION:CHARACTER_KNOWLEDGE -->
- 作者已知信息、batch brief 信息、source map 信息，不自动等于角色当下已知信息
- 角色台词、`os`、动作说明和镜头说明，都只能使用当前场内已经听到、看到、被明示或按关系天然知道的信息
- 只听到名，不得主动补姓；只见到脸，不得直接说全名；只知道“她 / 那位小姐 / 那个人”，不得偷换成更具体的身份称谓
- 原著若把姓名、身份、关系词拆成前后两步解锁，剧本也必须按原顺序解锁；不得把后段信息偷跑到前段台词里
- 对角色知识边界拿不准时，优先使用更中性的称谓与回应，不替角色抢答
- 回应句必须接住上一句真实触发词；上一句没出现的称谓、关系词和指控，不得凭作者脑内上下文自行补全

# 成稿风格（Writer Style）
<!-- PROMPT-REFERENCE: Writer prompt may reference stable style sections below. Keep section ids stable when editing. -->

本文件承接 Harness V2 下 writer 的具体写法风格。
`write-contract.md` 定义边界与硬门槛，本文件定义稳定成稿时应遵守的叙事姿态、场景打法和风格警戒。
当某条规则既涉及结构底线又涉及风格执行时，先服从 `write-contract.md`，再在其边界内做风格填充。

## 商业叙事目标（Runtime-Target Vertical Drama）
<!-- SECTION:COMMERCIAL_VERTICAL_DRAMA -->
- 当前目标总时长以 `run.manifest.md` 的 `target_total_minutes` 为准；不是 80-100 集投流拉长模型
- 推荐整体体感由 `target_total_minutes / target_episode_minutes` 估算，再由原著有效戏剧单元校正；信息密度优先于固定字数
- 每集都要有明确主冲突、行动推进或关系位移，不能只是原著事件摘录
- 每 3-5 集应形成一个小爽点或阶段性释放，每 8-10 集应形成一个大反转、大打脸或关系重排
- 爽点不是只写狠话；爽点必须落在身份、位置、证据、资源、公开场合、人物选择或不可逆后果上
- 保留短剧的直给情绪，但避免全员持续吼叫、全员金句、全员脸谱化降智
- 对外成稿必须使用剧名，不使用原著人物名；改名后的称呼要自然、统一、易记，不能像占位符

## 黄金开场（Golden Opening）
<!-- SECTION:GOLDEN_OPENING -->
- 前 3 集按“黄金三秒”标准处理：开场第一拍就给异常、羞辱、误认、逼选、抓捕、证据、身份错位或现场阻力
- EP-01 禁止从日常铺垫、慢环境、人物小传开始；前 10-20 秒必须让观众知道“谁被逼到什么局里”
- EP-01 至少完成“异常入局 + 第一轮关系冲突 + 强钩”，不能只停在“刚见面 / 刚到门口 / 即将开始”
- EP-02 和 EP-03 不能复述 EP-01 的冲突，应在开场立刻升级：更公开的羞辱、更硬的证据、更逼近的选择、更强的站位变化
- 前 3 集的集尾必须是付费短剧式强钩：身份即将揭露、反击即将发生、关系被迫锁死、证据被抛出或人物必须当场选择
- 黄金开场不等于乱加大事件；所有冲突仍必须来自 source 顺序和当前集边界

## 叙事姿态（Narrative Posture）
<!-- SECTION:NARRATIVE_POSTURE -->
- 镜头贴着人走，不做上帝视角俯瞰
- `△` 只写摄影机能拍到的空间、动作、姿态、道具与光线
- 第一人称原著进入剧本时，默认拆掉“我”叙述腔，改成镜头可拍的第三人称呈现
- 叙事温度偏冷，只给事实和动作，不替观众总结情绪
- “冷”不等于多说冷话；冷体现在少解释、少评断、少替观众下结论
- 观众知道多少取决于镜头对准了什么，没拍到的就不要替观众补
- `os` 是唯一主观窗口，只承担角色一闪而过的判断、代价感或旧记忆触发
- `♪` 不是装饰，而是节奏、停顿、压迫感和转场的一部分
- 禁止使用“他感到 / 她觉得 / 气氛变得 / 令人”式情绪总结句替代可拍内容
- 环境描写必须绑在当前人物所在位置、动作或视线里，不写脱离人物落点的俯瞰式铺陈
- 环境不写成散文串烧；同一拍只留 1-2 个最有效的视觉 / 听觉符号，不把晨光、白气、残滴、风声并排堆满

## 场景打法（Scene Craft）
<!-- SECTION:SCENE_CRAFT -->
- 每场先立环境，开场前两条 `△` 内至少要让观众看到空间、光线或温度
- `△` 必须包含可见动作、姿态变化或物理细节，不能写成散文摘要
- 场景不能只剩对白，对话之间要穿插动作、表情、反应和位置变化
- 每场都要想清观众听到什么，用 `♪` 标记环境声、配乐转折或刻意沉默
- `♪` 和 `【镜头】` 只在真的承担节奏转折、视角切换或信息强调时出现；不要把它们当段落填充物
- 多数场结尾不停在平地，优先留下未完成动作、新威胁、反应停顿或前推力
- 允许安静收尾，但它必须是刻意的节奏对比，不是没写满
- 施救、守夜、观察、潜伏、跟踪类场景允许低对白；推进可主要由动作、停顿、视线、道具和空间压迫承担
- 没有天然交锋的场景，不要硬写成来回斗嘴；能用动作推进，就不要先用狠话推进
- 试探场优先写“谁先察觉不对、谁更怕被看穿”；先保护不对称的压力，再决定是否需要对话来回
- “对视”“平静离开”“沉默站住”这类收尾，只有在前一拍已形成新判断、新威胁或未完成动作时才成立
- 证据戏不能只写投屏、文件、录音或资料宣读。每条关键信息至少绑定一个现场动作：谁按下遥控器、谁拦警察、谁夺文件、谁被迫签字、谁退后、谁被扣手铐、谁的权限被冻结
- 屏幕文字只负责证明事实，不负责替代冲突；真正的戏落在现场人物的选择、站位变化和后果承受上
- 法律、商业、旧案清算场要写物理流程：封门、收手机、冻结权限、交接文件、警员动作、账户状态跳变、手铐、签字、门禁失效等

## 对话打法（Dialogue Craft）
<!-- SECTION:DIALOGUE_CRAFT -->
- 有对抗的对话必须有推挡和交锋，不是角色轮流把信息说完
- 短剧对白可以比小说更直给，羞辱、质问、拒绝、揭露、反击要让观众一听就懂
- 直给不等于口号化；主角反击必须有事实、行动或局面变化托底，不能只靠“你们会后悔”式宣言
- 低对白源场和非对抗场，不强制制造来回交锋；允许人物用少量处理句、判断句和停顿完成推进
- 原著若只有一轮短试探，不扩写成多轮连续问答；尤其避免把问姓名、问师承、问祖籍、问父母拆成流水线式盘问
- 单句尽量短，但短句不是碎句排版；一个完整语义单元优先写在同一行
- 连续纯对白不超过 4 句，随后必须插入动作、反应或镜头转换
- `os` 不重复画面和台词已经给出的信息，删掉后仍看得懂的 `os` 应删
- `os` 最有效的用途是表里不一、代价感、旧记忆触发，而不是宣布“我赢了”
- 角色越想要某件事，越不该把真正目的直接说透
- 如果删掉 `os` 后整场动机就站不住，优先补潜台词和动作，不靠 `os` 兜底
- 不得连续抛出只用于“立冷”“立狠”“立聪明”的台词；一场里若这类句子连着出现，应主动删减
- 能用动作、停顿、视线和道具接句时，优先不用模板机锋句接句
- 关系宣告句必须晚于关系事实。人物还没演到那一步时，不替他提前把亏欠、信任、心软或占有说出口
- 对试探与疑心场，不要太早写成双方旗鼓相当的“高手过招”；若原著核心是谁心里更紧，就让镜头更多贴那个人
- 提到有重量的姓氏、名号、旧称或身份时，不只写“分量很重”；要让观众立刻感觉到它可能牵出的人、位置或后果

## 单集节奏（Episode Rhythm）
<!-- SECTION:EPISODE_RHYTHM -->
- 每集至少 2 场；按功能完成决定场数，通常 3-5 场，不为卡场数牺牲信息密度
- 每集必须有至少 1 个明确冲突、关系位移或反转
- 3-4 场时至少覆盖 2 种不同情绪；若场次更少，场内必须显出明显情绪转折
- 集尾必须是强闭环或前推力，不能平着结束
- 允许轻场，但轻场必须承担呼吸、过渡、反差或蓄压功能
- 当前目标体量下，单集不要写成薄切片；一集至少跑完一个“进入 -> 对抗/变化 -> 结果或钩子”的主戏闭环

## 风格红线（Style Red Lines）
<!-- SECTION:STYLE_RED_LINES -->
- 单场 `"像 / 仿佛 / 如同 / 好似"` 式明喻不超过 2 次，单集不超过 5 次
- `△` 禁止解释动机、分析性格或替观众判断
- 角色台词必须服从 `voice-anchor.md` 或 `character.md` 的声纹和禁用表达
- 遮住名字读台词时，核心角色之间必须能区分
- 不把复仇、宫斗或权谋写成纯斗嘴，规则、位置、名分和后果必须可见
- 不为撑节奏主动补模板狠话、冷句、机锋句或关系宣告句
- 遮住角色名后，若台词能无损塞进另一部古偶短剧，优先重写

## 角色知识边界（Character Knowledge Boundary）
<!-- SECTION:STYLE_CHARACTER_KNOWLEDGE -->
- 对话要服从角色当下视角，不服从作者全知视角
- 角色只听到名时，回应优先用“你认错人了 / 我不是她 / 这名字不属于我”之类的当下可知表达，不主动补姓
- 角色只看到脸、照片、背影、旧物时，优先写反应与停顿，不抢先说出全名
- 遇到误认、认亲、掉马前夜、替身错认类场景，先检查“角色现在究竟知道到哪一步”，再落台词
- 对知识边界拿不准时，宁可让角色少说一步，也不要替角色补完观众还没看到的信息

## 风格观察项（Style Watchlist）
<!-- SECTION:STYLE_WATCHLIST -->
- 同一微表情、体态词、核心意象不能在短窗口内高频复用
- 每集至少 1 条 `os` 不是工整金句
- 连续多场都用同一种抒情收尾时必须换打法
- 同一警戒项连续 3 集触发，需要在当前批次结束前回修
- 单场连续出现 3 句以上纯态度句、判断句而没有新动作承接时，应回头删句
- 低对白源场若改编后台词数明显膨胀，先删台词，再重整节奏
- 短试探若被写成长盘问、户口式问答或“信息盘查流程”，应回头压缩
- 若一场本应是单侧警惕，却被写成双方都很会说、都很会压的均衡对戏，应回头把镜头压回更危险的那一侧
- 同一场若连续堆叠多个 `♪` / `【镜头】`，却没有新增信息或节奏变化，应回头删减
- 开场环境若能删掉一半氛围细节而信息不受损，说明修饰密了，应回头压缩

## 输出姿态（Output Posture）
<!-- SECTION:OUTPUT_POSTURE -->
- 正文要像可拍的剧本，不像提纲、评论或小说旁白
- `△` 只呈现镜头所见
- 不把“我推门进去 / 我看见 / 我心里一沉”这类小说第一人称叙述直接搬进剧本正文
- `os` 是人的念头，不是金句海报，也不是剧情总结
- 对话应先让角色成立，再让信息成立；不是每一句都要显得“有劲”

# Quality Anchor

## 场景厚度

初始化同步：每集优先承担独立戏剧动作，不靠重复羞辱、重复试探、重复态度词填充时长。

## 对话节奏

初始化同步：对白优先承接原著关系温度与信息顺序；短句服务情境，不服务模板化网感。

## os 使用方式

初始化同步：os 只补瞬时判断或代价感，不替代剧情说明，不重复画面已给信息。

## 代表性打法
- 推荐区间：20-32 集。按 50 分钟目标、单集 1-3 分钟浮动、中心 2 分钟估算，25 集是当前最稳的商业短剧长度；既能保留错认、断亲、掉马、护妻、断亲求和、养女清算、姐姐归来、幕后黑手、终极反转、求婚婚礼等高价值节点，又不会拉成长链条水集。
- 最终采用：25 集。按 5 个 batch 组织，每批 5 集。前 5 集完成“错认入局、真千金断亲、身份掉马、男主护短、父母求和失败”的免费强钩子段；第 6-10 集推进追妻和养女反扑并引出姐姐；第 11-15 集完成姐妹相认与温绮罗第一次清算；第 16-20 集推进旧案、钟启衡、假助理栽赃和最终证据反打；第 21-25 集完成恶人终局、求婚、姐夫回归和双婚礼情绪闭环。
- 可独立成集戏剧节点：街头错认强行带走；温家初见被盘问；亲子鉴定确认后断亲；晚宴顶级设计师掉马；顾沉舟当众护她并与温家撕破脸；温家上门求和被拒；顾沉舟追到公寓被设边界；温绮罗设计陷害姜照眠；姜照眠反手公开证据；温知澜归来；姐妹相认但互相试探；温绮罗被逐出温家；温绮罗反扑嫁祸；姐妹联手反击；姜照眠事业高光；旧案线索浮现；钟启衡身份暴露；三方收网；假助理指控温知澜；温知澜反证温绮罗；温绮罗彻底落网；温家父母真心悔罪但姜照眠保留边界；顾沉舟求婚；秦砚白回归确认温知澜幸福；双人婚礼大团圆。
- 应合并压缩的内容：重复羞辱、重复“你不如养女”、重复求和、重复追妻日常、重复资料查证、重复屏幕证据展示、纯甜宠铺垫、婚礼前过长抒情都要压缩。每一集必须有新结果：关系改变、身份暴露、阵营重排、证据落地、权力转移或不可逆选择。
- 为什么不是更短/更长：少于 20 集会压掉姐姐归来、幕后黑手和终极反转，导致后半段只剩速通；超过 32 集会把原著有限冲突拆成重复求和、重复追妻和重复证据屏幕，降低信息密度。25 集能覆盖全部有效戏剧单元，并保持每 3-5 集一个释放点、每 8-10 集一个大反转。前 3 集黄金开场应处理为：EP-01 前 3 秒直接街头拦人和错认，不做生活铺垫；EP-02 亲子鉴定确认后父母继续羞辱，姜照眠主动断亲；EP-03 晚宴顶级设计师身份公开，直接打脸温家。阶段性爽点落点：EP-03 掉马，EP-05 求和失败，EP-09 反陷害，EP-12 姐姐归来，EP-15 养女第一次出局，EP-19 假助理栽赃大反转，EP-20 终极反打，EP-23 求婚，EP-25 婚礼闭环。

# Voice Anchor

用于锁定关键角色的说话方式，防止全员同腔。

  ## 使用原则
  - Writer 优先读取本文件；未列出的角色再回退到 `character.md`
  - 若本文件只剩模板占位，视为未填写，直接回退到 `character.md`
  - 没有原著时，只保留语音指纹；有原著时再补"原著样句 + 抽象特征"
  - 这里只存说话方式，不复述人物经历
  - 外显台词和 `os` 分开看：外显更克制，`os` 可以更短、更狠，但不能全员金句化
  - 优先写“温度、节奏、回避/施压方式、面对不同对象时的变化”，不要写成口号库
  - 除非原著反复出现固定说法，否则不要整理固定句式清单

  ## 格式警告（所有角色通用）
  本文件描述的是**角色说话内容的特征**（简洁、压迫、快节奏等），不是剧本排版指令。
  - "短句"＝角色说话简洁利落，不等于每句话都要单独占一行
  - "连打"/"连下两三句"＝角色一口气连续逼问或施压，通常写在同一行或相邻两行，不是拆成五六行
  - 只有语义上有**刻意停顿、转折、施压节拍**时才换行
  - 具体格式规则见 `write-contract.md` 的 Dialogue Formatting 章节

  ## 核心角色

### 姜照眠
- 说话温度：外冷内稳，不急着证明自己。面对羞辱时先接住对方话里的漏洞，再轻轻反推。
- 句法与节奏：短句多，判断句多，少解释。关键反击要像落章，一句落地后让现场停住。
- 对不同对象的变化：对温家父母克制疏离；对温绮罗直接拆戏；对顾沉舟先设边界，后期才允许柔软；对温知澜会出现更轻的迟疑。
- 外显台词边界：不喊口号，不长篇诉苦，不用网络爽文腔堆叠身份。
- `os` 边界：可以冷讽和自我判断，但不能频繁内心独白解释剧情。
- 禁止滑坡：不要把她写成全程开挂嘲讽机器；她的厉害要通过行动和选择呈现。

### 顾沉舟
- 说话温度：对外低温压迫，对姜照眠从命令式逐步变成克制保护。
- 句法与节奏：前期多确定句和命令句；中后期在姜照眠面前会主动收短、留余地。
- 对不同对象的变化：对温家是直接切断利益的上位者；对温绮罗不接情绪表演；对温知澜有旧情尊重但边界清楚；对姜照眠不能一直霸总命令。
- 外显台词边界：可以强势护短，但别把每句话都写成“全城封杀”。
- `os` 边界：少用内心悔恨大段独白，更多让他通过停手、退后、道歉和护场体现变化。
- 禁止滑坡：不能同时对两姐妹暧昧，不能用追求包装控制。

### 温知澜
- 说话温度：平静、温柔、有预判。被栽赃时不慌，反证时不吼。
- 句法与节奏：先留白，再递证据；重要信息一层一层揭，不一次倒完。
- 对不同对象的变化：对姜照眠柔和且有亏欠；对温绮罗冷静拆穿；对顾沉舟礼貌疏离；对秦砚白会露出难得的放松。
- 外显台词边界：不要写成圣母姐姐，也不要只负责解释资料。
- `os` 边界：少用；她的心理应通过停顿、护住妹妹、按下遥控器等动作表达。
- 禁止滑坡：不能把她写成只会“妹妹别怕”的工具人。

### 温绮罗
- 说话温度：前期软、湿、会退让；后期尖、急、破功。
- 句法与节奏：前期常用半句、停顿、把责任往自己身上揽但实际引导别人攻击姜照眠；后期语速变快，质问变多。
- 对不同对象的变化：对梁淑仪撒娇求护；对温正廷装懂事；对姜照眠表面道歉实际挑衅；对钟启衡和假助理会暴露命令感。
- 外显台词边界：不要一上来疯批化，她要先“看起来善良”。
- `os` 边界：可以暴露嫉妒和算计，但不要让她提前把全部阴谋自白。
- 禁止滑坡：不要只会哭；她必须会推动事件。

### 温正廷
- 说话温度：端着父权和家族体面，真正失控时才气急。
- 句法与节奏：盘问式、命令式，喜欢用“规矩、体面、温家”压人。
- 对不同对象的变化：对姜照眠审视；对温绮罗纵容；对顾沉舟忌惮；被打脸后语气从硬变虚。
- 外显台词边界：不能太早真诚道歉。
- `os` 边界：不需要多。
- 禁止滑坡：不要写成纯小丑，他的势利要具体落在利益和脸面上。

### 梁淑仪
- 说话温度：情绪化、尖、护短，对姜照眠容易先责备后后悔。
- 句法与节奏：容易连珠炮，常把“懂事、感恩、别让我们难堪”挂在逻辑里。
- 对不同对象的变化：对温绮罗柔软心疼；对姜照眠挑剔；求和时声音会虚、会急。
- 外显台词边界：不要把她悔意写成一句哭就解决。
- `os` 边界：少用。
- 禁止滑坡：不能突然完全明事理，她的转变要被事实逼出来。

### 钟启衡
- 说话温度：阴冷、威胁感强，自以为掌控全局。
- 句法与节奏：威胁句、反问句、旧怨陈述多；现场败露后会从镇定转向狂躁。
- 对不同对象的变化：对姐妹是毁灭性威胁；对温绮罗有利用和轻视；被反利用后破防。
- 外显台词边界：不要大段解释旧案，旧案通过证据和对峙分层露出。
- `os` 边界：不写或极少。
- 禁止滑坡：不能替终极反派说完所有真相。

### 秦砚白
- 说话温度：温和但有底气，保护欲不张扬。
- 句法与节奏：句子不长，更多是确认、承诺、接住温知澜。
- 对不同对象的变化：对温知澜宠溺；对温家礼貌；对顾沉舟平等对视；对外人有压迫感但不抢话。
- 外显台词边界：不要突然变成另一个霸总模板。
- `os` 边界：基本不需要。
- 禁止滑坡：只补姐姐线，不要扩写成新主线。

## 核心人物

### 姜照眠
- 功能定位：真千金女主、顶级设计师“眠光”、被错认后卷入温家旧案的人。
- 处境压力：被陌生权势男人错认、被亲生父母盘问嫌弃、被养女挑拨占位；她拥有事业底牌，但一开始不主动炫耀。
- 欲望：保持自我命名权，不被血缘、替身关系、豪门评价重新定义。
- 软肋：对亲情曾有一丝期待；面对姐姐线时更容易动摇。
- 外在抓手：手稿、高定面料、限量腕表、冷静收回手、把证据推到桌面、公开场合不躲目光。
- 行动边界：可以冷，可以狠，但不是无差别刻薄；她的强来自判断和选择，不来自大吼。

### 顾沉舟
- 功能定位：男主、豪门掌权人、从错认强控到真正追求的护短者。
- 处境压力：失去温知澜的执念让他开局越界；越了解姜照眠，越必须承认自己认错并修正行为。
- 欲望：先是找回失去的人，后是守住眼前真实的人。
- 软肋：失而复得的幻觉、对姜照眠受辱时的失控保护欲。
- 外在抓手：黑色西装、压低视线、挡在她身前、一个眼神让全场安静、签停合作令。
- 行动边界：强势但不能抢走姜照眠的主导权；追求不是命令，护短不是替她原谅别人。

### 温知澜
- 功能定位：失踪姐姐、后半段并肩女主、旧案真相的关键持证人。
- 处境压力：长期被失踪、死亡、阴谋包裹；回归后还要承受“可能是真凶”的栽赃。
- 欲望：保护妹妹，清算旧案，拿回自己和温家的真相。
- 软肋：对妹妹的亏欠感，对父母多年失衡的复杂情绪。
- 外在抓手：文件袋、海外证据、遥控器点亮大屏、先沉默后反证。
- 行动边界：温柔不是软弱；她要有预判和布网能力，不能只当资料讲解员。

### 温绮罗
- 功能定位：养女反派、白莲占位者、终极操盘者。
- 处境压力：姜照眠归来威胁她长期占有的身份、资源和父母偏爱。
- 欲望：保住温家大小姐位置，夺走资源，把真姐妹踩下去。
- 软肋：嫉妒、身份不稳、无法接受“不是亲生却想占尽一切”的事实。
- 外在抓手：红眼、拽母亲衣角、柔弱道歉、背后删证据、失控尖叫。
- 行动边界：前期善用示弱，不要一上来疯；后期可以暴露狠毒和计划性。

### 温正廷
- 功能定位：偏心父亲、势利家长、迟来悔意的打脸对象。
- 处境压力：家族体面、商业利益、真假女儿冲突。
- 欲望：控制家庭秩序，维护温家面子和利益。
- 软肋：势利、怕丢脸、怕合作断裂。
- 外在抓手：拍桌、翻鉴定书、端坐主位、被合作方冷眼后站不稳。
- 行动边界：前期不能太快悔，悔意必须来自事实打脸和利益坍塌。

### 梁淑仪
- 功能定位：偏心母亲、情绪压迫源、养女保护伞。
- 处境压力：在亲女儿和养女之间选择时，她本能护住更会讨好的那个。
- 欲望：保住“体面母亲”的自我想象，同时继续享受养女的依赖。
- 软肋：情绪化、耳根软、被温绮罗牵动。
- 外在抓手：拉住养女、掉眼泪、伸手想碰姜照眠却被避开。
- 行动边界：不要突然圣母化；她的悔必须有羞愧和无力感。

### 钟启衡
- 功能定位：旧案阶段性黑手、外部威胁、被终极反派利用的重子。
- 处境压力：旧仇暴露、资产被冻结、退路被封。
- 欲望：夺回利益，毁掉温家，拖姐妹下水。
- 软肋：自以为掌控全局，实际被温绮罗借刀。
- 外在抓手：匿名短信、伪造股权文件、会议室闯入、被手铐扣住前反咬。
- 行动边界：他负责把家庭伦理戏推成危险收网戏，但不能覆盖温绮罗的终极反转。

### 秦砚白
- 功能定位：温知澜丈夫、后段情绪补偿和保护力量。
- 处境压力：长时间在海外陪温知澜查证，回归时主线已接近收束。
- 欲望：带温知澜回到安全、公开、被祝福的位置。
- 软肋：对温知澜的心疼。
- 外在抓手：逆光入场、伸手接人、礼貌但有压迫感地对外确认身份。
- 行动边界：不抢顾沉舟和姜照眠主线，只补足姐姐线。
