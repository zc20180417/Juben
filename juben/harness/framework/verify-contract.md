# Verify Contract

本文件定义 verify 阶段的阻断标准。

## Verify Inputs
- draft lane 候选正文
- `_ops/episode-lint.py` 输出
- `harness/project/regressions/` 下的活跃回归 pack（如存在）
- 项目层 state 文件
- `voice-anchor.md`（优先）→ 回退 `character.md`
- `quality.anchor.md`（如存在）

## Fail Closed
- draft 未通过 `_ops/episode-lint.py`
- 回应句失去触发语义
- 原著自然事件被无依据改成设计局
- 原著概述性信息被擅自扩成具体事故
- 原著强闭环被只为造钩子硬切断
- 即时动作钩子在相邻下一集第一场未兑现
- draft 直写 publish lane

## Warning
- 单点关系稀释：单人第一次注意/记住的场被扩成群像热闹场
- `os` 语义悬空：读者无法直接读清主语、对象或判断落点
- 机械碎句化
- `os` 功能错位

## Escalation
- 同一集出现 2 处及以上 `os` 语义悬空：升级为 FAIL
- 单点关系被明显冲淡到改变场景核心功能：直接 FAIL
- 连续 3 集出现同类 warning：当前批次不得 promote

## Verify Checklist

以下检查在 lint 机械检查通过后执行。

### 对抗性检查（EP-XX 必执行）
1. **角色替换测试**：抽取角色A的对话假设换成角色B说，毫无违和感 → FAIL
2. **删除测试**：假设删掉某场景，剧情完全不受影响 → FAIL
3. **逻辑反推测试**：从场景结果反推原因，找不到前文因果链 → FAIL
4. **钩子有效性测试**：读完集尾最后一段，没有"接下来会怎样"的疑问 → FAIL
5. **画面感测试**：抽取△描写，闭眼无法想象具体画面 → FAIL；追加摄影机测试：无法用镜头呈现 → 作者评论混入 → FAIL
6. **表里不一测试**（仅限有 os 角色）：
   - os 与对话无反差（只是重复对话意思）→ os 无效 → FAIL
   - 删掉 os 后观众仍然看得懂 → os 多余 → WARNING（同集累计 3 条 → FAIL）
   - 删掉全部 os 后观众完全看不懂角色在做什么 → 台词潜台词不足 → FAIL

### 声纹硬查
- 声纹基准来源：优先 `voice-anchor.md`，缺失或未列当前角色时回退 `character.md`
- **句长分布检查**：短句型角色单句超 15 字台词不超过 10%；长句型角色以定性检查为主；直来直去型角色不应出现迂回暗示句式
- **样本量门槛**：角色本集有效台词 ≥ 5 句时执行句长分布硬查；不足时只做定性抽查
- **角色区分度测试**：每集抽取至少 2 组不同角色台词段（各 3-5 句），遮盖角色名后无法区分说话人 → FAIL
- **禁用表达检查**：声纹基准中标注的禁用表达出现即 FAIL
- **原著声纹锚**（有原著时）：抽查角色台词是否同时符合原著样句气质与抽象特征，保持称呼习惯、拐弯程度、常用词一致性

### 质量锚对标（quality.anchor.md 存在时执行）
- 最低丰满度契约全部通过后才进入质量锚对标
- 读取 `quality.anchor.md`，将最近一批（默认 5 集）与锚点对比
- 核心对比维度：
  - 场景厚度是否明显变薄
  - os 是否从"暴露真实意图"退化成"重复表面意思"
  - 表情/镜头/动作是否只剩模板化点缀
  - 场尾钩子是否失去冲击力
  - 角色台词是否越来越像同一个人
- 最近一批中有 3 集及以上明显弱于锚点 → 批量模式 FAIL

### 表情与密度检查
- 表情/微表情特写：每集 ≥ 2 处，不足即 FAIL
- 冲突/反转：每集 ≥ 1 个，必须能明确指出是哪个
- 情绪层次：2-3 场时至少 2 个不同主情绪；1 场时场内必须有明确情绪转折

### 对白语义漂移检查（preserve / light 档）
- 台词不得新增原著没有的态度、挑衅、机锋、判断或更聪明的反击
- 命中后必须指出漂移场次、对应原著语义、漂移类型
- 漂移类型：态度增强 / 机锋增强 / 关系温度偏移 / 说明化
