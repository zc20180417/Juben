# 成稿风格（Writer Style）
<!-- PROMPT-REFERENCE: Writer prompt may reference stable style sections below. Keep section ids stable when editing. -->

本文件承接 Harness V2 下 writer 的具体写法风格。
`write-contract.md` 定义边界与硬门槛，本文件定义稳定成稿时应遵守的叙事姿态、场景打法和风格警戒。
当某条规则既涉及结构底线又涉及风格执行时，先服从 `write-contract.md`，再在其边界内做风格填充。

## 商业叙事目标（Runtime-Target Vertical Drama）
<!-- SECTION:COMMERCIAL_VERTICAL_DRAMA -->
- 当前目标总时长以 `run.manifest.md` 的 `target_total_minutes` 为准；不是 80-100 集投流拉长模型
- 推荐整体体感由 `target_total_minutes / target_episode_minutes` 估算，再由原著有效戏剧单元校正；信息密度优先于固定字数
- 每集都要有明确主冲突、行动推进或关系位移，不能只是原著事件摘录
- 每 3-5 集应形成一个小爽点或阶段性释放，每 8-10 集应形成一个大反转、大打脸或关系重排
- 爽点不是只写狠话；爽点必须落在身份、位置、证据、资源、公开场合、人物选择或不可逆后果上
- 保留短剧的直给情绪，但避免全员持续吼叫、全员金句、全员脸谱化降智
- 对外成稿必须使用剧名，不使用原著人物名；改名后的称呼要自然、统一、易记，不能像占位符
- 改名不是原创化改编的终点。剧本必须重构原作外壳：行业/职业、场景空间、关键道具、证据形态、事件触发方式和冲突载体都应服务新剧设定
- 原著只提供叙事功能和情绪曲线，不提供可直接复用的台词、镜头、场面模板或特殊道具
- 如果一场戏只需把剧名换回原名就能还原原作场景，这场戏应重写为新设定下的可拍行动

## 黄金开场（Golden Opening）
<!-- SECTION:GOLDEN_OPENING -->
- 前 3 集按“黄金三秒”标准处理：开场第一拍就给异常、羞辱、误认、逼选、抓捕、证据、身份错位或现场阻力
- EP-01 禁止从日常铺垫、慢环境、人物小传开始；前 10-20 秒必须让观众知道“谁被逼到什么局里”
- EP-01 至少完成“异常入局 + 第一轮关系冲突 + 强钩”，不能只停在“刚见面 / 刚到门口 / 即将开始”
- EP-02 和 EP-03 不能复述 EP-01 的冲突，应在开场立刻升级：更公开的羞辱、更硬的证据、更逼近的选择、更强的站位变化
- 前 3 集的集尾必须是付费短剧式强钩：身份即将揭露、反击即将发生、关系被迫锁死、证据被抛出或人物必须当场选择
- 黄金开场不等于乱加大事件；所有冲突仍必须来自 source 顺序和当前集边界

## 叙事姿态（Narrative Posture）
<!-- SECTION:NARRATIVE_POSTURE -->
- 镜头贴着人走，不做上帝视角俯瞰
- `△` 只写摄影机能拍到的空间、动作、姿态、道具与光线
- 第一人称原著进入剧本时，默认拆掉“我”叙述腔，改成镜头可拍的第三人称呈现
- 叙事温度偏冷，只给事实和动作，不替观众总结情绪
- “冷”不等于多说冷话；冷体现在少解释、少评断、少替观众下结论
- 观众知道多少取决于镜头对准了什么，没拍到的就不要替观众补
- `os` 是唯一主观窗口，只承担角色一闪而过的判断、代价感或旧记忆触发
- `♪` 不是装饰，而是节奏、停顿、压迫感和转场的一部分
- 禁止使用“他感到 / 她觉得 / 气氛变得 / 令人”式情绪总结句替代可拍内容
- 环境描写必须绑在当前人物所在位置、动作或视线里，不写脱离人物落点的俯瞰式铺陈
- 环境不写成散文串烧；同一拍只留 1-2 个最有效的视觉 / 听觉符号，不把晨光、白气、残滴、风声并排堆满

## 场景打法（Scene Craft）
<!-- SECTION:SCENE_CRAFT -->
- 每场先立环境，开场前两条 `△` 内至少要让观众看到空间、光线或温度
- `△` 必须包含可见动作、姿态变化或物理细节，不能写成散文摘要
- 场景不能只剩对白，对话之间要穿插动作、表情、反应和位置变化
- 每场都要想清观众听到什么，用 `♪` 标记环境声、配乐转折或刻意沉默
- `♪` 和 `【镜头】` 只在真的承担节奏转折、视角切换或信息强调时出现；不要把它们当段落填充物
- 多数场结尾不停在平地，优先留下未完成动作、新威胁、反应停顿或前推力
- 允许安静收尾，但它必须是刻意的节奏对比，不是没写满
- 施救、守夜、观察、潜伏、跟踪类场景允许低对白；推进可主要由动作、停顿、视线、道具和空间压迫承担
- 没有天然交锋的场景，不要硬写成来回斗嘴；能用动作推进，就不要先用狠话推进
- 试探场优先写“谁先察觉不对、谁更怕被看穿”；先保护不对称的压力，再决定是否需要对话来回
- “对视”“平静离开”“沉默站住”这类收尾，只有在前一拍已形成新判断、新威胁或未完成动作时才成立
- 每集至少设置 1 个与当前主戏相关的现场阻力：人物阻拦、空间限制、资源被卡、时间压力、身份不被承认、情绪不配合、规则/名分/权限限制、证据被夺或被质疑都可以
- 现场阻力必须来自当前 source 和当前 beats，不为制造波折新增 source 外大事件；轻场可以用门不开、手抽回、礼物被退、座位排挤、沉默拒绝、对方后退等低强度阻力
- 结论、证据、告白、道歉、认亲、交易或反击不要一出现就自动成功；至少先给一个可拍的抵抗、选择或代价，再落结果
- 不要把“现场阻力”理解成每集都要反派吵架或证据翻盘；不同题材应落成不同动作：情感戏看边界，家族戏看名分和站位，商战/法律戏看流程和权限，危险戏看空间和时间
- 证据戏不能只写投屏、文件、录音或资料宣读。每条关键信息至少绑定一个现场动作：谁按下遥控器、谁拦警察、谁夺文件、谁被迫签字、谁退后、谁被扣手铐、谁的权限被冻结
- 屏幕文字只负责证明事实，不负责替代冲突；真正的戏落在现场人物的选择、站位变化和后果承受上
- 法律、商业、旧案清算场要写物理流程：封门、收手机、冻结权限、交接文件、警员动作、账户状态跳变、手铐、签字、门禁失效等

## 对话打法（Dialogue Craft）
<!-- SECTION:DIALOGUE_CRAFT -->
- 有对抗的对话必须有推挡和交锋，不是角色轮流把信息说完
- 短剧对白可以比小说更直给，羞辱、质问、拒绝、揭露、反击要让观众一听就懂
- 直给不等于口号化；主角反击必须有事实、行动或局面变化托底，不能只靠“你们会后悔”式宣言
- 低对白源场和非对抗场，不强制制造来回交锋；允许人物用少量处理句、判断句和停顿完成推进
- 原著若只有一轮短试探，不扩写成多轮连续问答；尤其避免把问姓名、问师承、问祖籍、问父母拆成流水线式盘问
- 单句尽量短，但短句不是碎句排版；一个完整语义单元优先写在同一行
- 连续纯对白不超过 4 句，随后必须插入动作、反应或镜头转换
- `os` 不重复画面和台词已经给出的信息，删掉后仍看得懂的 `os` 应删
- `os` 最有效的用途是表里不一、代价感、旧记忆触发，而不是宣布“我赢了”
- 角色越想要某件事，越不该把真正目的直接说透
- 如果删掉 `os` 后整场动机就站不住，优先补潜台词和动作，不靠 `os` 兜底
- 不得连续抛出只用于“立冷”“立狠”“立聪明”的台词；一场里若这类句子连着出现，应主动删减
- 能用动作、停顿、视线和道具接句时，优先不用模板机锋句接句
- 关系宣告句必须晚于关系事实。人物还没演到那一步时，不替他提前把亏欠、信任、心软或占有说出口
- 对试探与疑心场，不要太早写成双方旗鼓相当的“高手过招”；若原著核心是谁心里更紧，就让镜头更多贴那个人
- 提到有重量的姓氏、名号、旧称或身份时，不只写“分量很重”；要让观众立刻感觉到它可能牵出的人、位置或后果

## 单集节奏（Episode Rhythm）
<!-- SECTION:EPISODE_RHYTHM -->
- 每集至少 2 场；按功能完成决定场数，通常 3-5 场，不为卡场数牺牲信息密度
- 每集必须有至少 1 个明确冲突、关系位移或反转
- 每集至少有 1 个现场阻力或行动阻力，阻力强度按题材和当前 beats 调整，不允许关键结果无阻力地自动达成
- 3-4 场时至少覆盖 2 种不同情绪；若场次更少，场内必须显出明显情绪转折
- 集尾必须是强闭环或前推力，不能平着结束
- 允许轻场，但轻场必须承担呼吸、过渡、反差或蓄压功能
- 当前目标体量下，单集不要写成薄切片；一集至少跑完一个“进入 -> 对抗/变化 -> 结果或钩子”的主戏闭环

## 成稿前自我打磨（Agent Polish）
<!-- SECTION:AGENT_POLISH -->
- Writer 写入文件前必须做一轮自我打磨；这是表达层打磨，不新增剧情、不改变 source 顺序、不偷后续 payoff
- 删除或改写解释句：凡是“说明动机 / 总结情绪 / 替观众判断 / 复述资料含义”的句子，优先改成动作、站位、物件、沉默、停顿或现场后果
- 压缩功能性台词：角色不要轮流把资料说完；每段核心信息后至少给一个现场反应、阻拦、选择或位置变化
- 每个核心场保留 1-2 个能拍出来的压力点：门被挡住、手抽回、文件被按住、座位变化、警员停手、手机反扣、屏幕暂停、人物后退或沉默不答
- 声纹打磨优先于金句打磨；让姜照眠更冷静克制、顾沉舟更低温留边界、温知澜更平静有预判、反派更会利用现场，不要把所有人写成同一种狠话节奏
- 证据、资料、屏幕和录音只能证明事实；打磨时必须补足“谁因此被迫选择 / 谁失去位置 / 谁被当场拦下 / 谁不能再沉默”
- 如果某场删掉一半台词后主戏仍成立，优先删台词并补动作；如果删掉动作后只剩资料宣读，必须回头重写场面

## 风格红线（Style Red Lines）
<!-- SECTION:STYLE_RED_LINES -->
- 单场 `"像 / 仿佛 / 如同 / 好似"` 式明喻不超过 2 次，单集不超过 5 次
- `△` 禁止解释动机、分析性格或替观众判断
- 角色台词必须服从 `voice-anchor.md` 或 `character.md` 的声纹和禁用表达
- 遮住名字读台词时，核心角色之间必须能区分
- 不把复仇、宫斗或权谋写成纯斗嘴，规则、位置、名分和后果必须可见
- 不为撑节奏主动补模板狠话、冷句、机锋句或关系宣告句
- 遮住角色名后，若台词能无损塞进另一部古偶短剧，优先重写

## 角色知识边界（Character Knowledge Boundary）
<!-- SECTION:STYLE_CHARACTER_KNOWLEDGE -->
- 对话要服从角色当下视角，不服从作者全知视角
- 角色只听到名时，回应优先用“你认错人了 / 我不是她 / 这名字不属于我”之类的当下可知表达，不主动补姓
- 角色只看到脸、照片、背影、旧物时，优先写反应与停顿，不抢先说出全名
- 遇到误认、认亲、掉马前夜、替身错认类场景，先检查“角色现在究竟知道到哪一步”，再落台词
- 第一次喊出对方姓氏、职位、关系称谓或身份名号前，必须先让观众看到来源；没有来源就写成中性称呼或直接用“你”
- 反击句要像真人顺口说出来：主谓宾完整，指向清楚，不为了“狠”写成夹生句
- 对知识边界拿不准时，宁可让角色少说一步，也不要替角色补完观众还没看到的信息

## 风格观察项（Style Watchlist）
<!-- SECTION:STYLE_WATCHLIST -->
- 同一微表情、体态词、核心意象不能在短窗口内高频复用
- 每集至少 1 条 `os` 不是工整金句
- 连续多场都用同一种抒情收尾时必须换打法
- 同一警戒项连续 3 集触发，需要在当前批次结束前回修
- 单场连续出现 3 句以上纯态度句、判断句而没有新动作承接时，应回头删句
- 低对白源场若改编后台词数明显膨胀，先删台词，再重整节奏
- 短试探若被写成长盘问、户口式问答或“信息盘查流程”，应回头压缩
- 若一场本应是单侧警惕，却被写成双方都很会说、都很会压的均衡对戏，应回头把镜头压回更危险的那一侧
- 同一场若连续堆叠多个 `♪` / `【镜头】`，却没有新增信息或节奏变化，应回头删减
- 开场环境若能删掉一半氛围细节而信息不受损，说明修饰密了，应回头压缩

## 输出姿态（Output Posture）
<!-- SECTION:OUTPUT_POSTURE -->
- 正文要像可拍的剧本，不像提纲、评论或小说旁白
- `△` 只呈现镜头所见
- 不把“我推门进去 / 我看见 / 我心里一沉”这类小说第一人称叙述直接搬进剧本正文
- `os` 是人的念头，不是金句海报，也不是剧情总结
- 对话应先让角色成立，再让信息成立；不是每一句都要显得“有劲”
