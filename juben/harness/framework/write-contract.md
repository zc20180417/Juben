# Write Contract

本文件定义 writer 的通用写作边界，不包含项目专有章节映射。

## Writing Posture
- `△` 只写摄影机能拍到的内容
- `os` 是主观窗口，但不能承担剧情讲解职责
- 完整气口优先于碎句节拍（见下方 Dialogue Formatting）

## Dialogue Rules
- 允许换说法，不允许换态度
- 不得把受压、求生、陈述改成挑衅、机锋、评判或聪明压人
- 保留回应句时，必须保留上句触发语义
- 不得把任何角色统一压成碎句机器（包括主角）

## Dialogue Formatting
"短句"指角色说话内容简洁，不是剧本排版上每行都要断开。区分方式：

- **一个语义单元 = 一行**。角色一口气说完的话，即使由几个短句组成，也写在同一行
- 只有在语义上有**停顿、转折、施压**时才换行——换行本身就是节奏工具
- 错误示范（机械断行）：
  ```
  萧景珩：让朕的嫔妃当众献舞。
  萧景珩：你当她是歌姬？
  ```
- 正确示范（语义换行）：
  ```
  萧景珩：让朕的嫔妃当众献舞，你当她是歌姬？
  ```
- 当角色**刻意施压、逼问、一句一句钉**时，才拆成多行：
  ```
  萧景珩：聚聚。
  萧景珩：聚到她掉眼泪？
  ```
  ↑ 这里"聚聚"是冷重复，第二句是反问施压——两句之间有刻意停顿，所以分行。
- 判断标准：如果拆开后两行之间**读不出停顿感**，就不该拆

## OS Rules
- `os` 必须一眼读清主语、对象和判断落点
- `os` 不得悬空到需要读者自己补半句
- `os` 不解释剧情，不提前宣布计划得手，不把观众判断说穿
- `os` 应优先承担：表里不一、代价感、被触发的旧记忆

## Scene Rules
- 每集 1-3 场
- 每集必须有明确戏剧推进
- 集尾要么有前推力，要么有强闭环落点
- 即时动作钩子只有在相邻下一集第一场会兑现时才允许使用

## Adaptation Posture
- 改编不是压缩字数，而是重分配信息载体
- 先保主线推进，再保关键关系变化，再保爆点与反转，最后保世界规则与主题表达
- 原著叙述信息优先外化成：动作、停顿、道具、环境变化、对话反应
- 角色心理先尝试外化成表情、身体动作、选择和说话方式；只有画面承载不了时才进入 `os`
- 原著里”写出来很好看”的段落，不等于搬成剧本就有效
- 剧本语言只保留人物会说的话、摄影机能拍到的事、剪辑能组织出的信息
- 原著叙述腔、优美句子、哲理句、旁白总结句不直接入正文

## Voice Anchor
- writer 优先读取 `voice-anchor.md`
- 若 `voice-anchor.md` 不存在、仅剩模板占位或未列出当前角色，回退到 `character.md` 中的语音指纹
- 只继承说话方式，不搬运小说叙述腔
- 角色台词必须符合声纹锚中的句长基准、句式基准和禁用表达

## Original-Fidelity Rules
- 章节顺序优先
- 一章可拆多集，多个相邻章节可合拍
- 不得抢跑后续章节大事件
- 原著自然发生的关键节点，不得无依据改成预谋布局
- 原著概述性信息，不得擅自扩成新的人名、具体事故或完整事件链
- 若原著某场核心是”某个人第一次单独注意到她 / 记住她 / 被她刺中”，不得扩成多人围观、多人评价的群像场
- 这类单点被看见、单点被记住、单点被刺中的关系场，优先保护其专属性
