# Adaptation Core

本文件只处理一件事：把小说信息转成短剧可拍的场景、动作、对白和镜头。
它不定义题材节奏，不定义平台付费节点，不定义具体爽点打法。

## 改编原则
- 先保主线，再保关键关系，再保爆点，最后才保原著枝节
- 原著里“写出来很好看”的段落，不等于搬成剧本就有效
- 观众只能看到镜头给出的东西：叙述、评价、议论都要外化
- 改编不是压缩字数，而是重分配信息载体

## 信息优先级
1. 主线推进
2. 关键关系变化
3. 爆点与反转
4. 世界规则
5. 主题表达

若素材冲突，优先保前 3 项；后两项通过台词、道具或场景功能补回，不强行全保。

## 叙述外化
- 原著叙述信息优先外化成：动作、停顿、道具、环境变化、对话反应
- 角色心理先尝试外化成表情、身体动作、选择和说话方式；只有画面承载不了时才进入 `os`
- 叙述性的总结句不要直接进剧本，要改成观众能看到的结果
- 原著里的解释段、分析段、议论段，不直接搬进 `△`

## 内心戏转换
- 适合转 `os` 的内容：
  - 与表面行为矛盾的念头
  - 一闪而过的代价感
  - 被触发的旧画面、旧话、旧伤
- 更适合转镜头或动作的内容：
  - 观众已经能从场面看懂的算计
  - 人物动机的完整解释
  - 作者视角的评论
- 如果删掉一条 `os` 后观众仍然完全看得懂，这条 `os` 应优先删除

## 压缩与合并
- 支线压缩：保功能，不保原始篇幅
- 角色合并：优先合并功能重复、情感权重低、只承担单次推动作用的人物
- 时间压缩：允许把原著中反复出现的同类事件合成一次高密度戏
- 不要为保留原著细节而牺牲场次功能和节奏

## 原著文风切断
- 原著叙述腔不直接进入剧本
- 小说里的优美句子、哲理句、旁白总结句，默认不入正文
- 剧本语言只保留人物会说的话、摄影机能拍到的事、剪辑能组织出的信息

## 改编自检
- 这条信息不靠原著旁白，观众还能看懂吗
- 这场戏承担的是主线、关系，还是爆点功能
- 这段原著内容是该压缩、合并，还是外化
- 这句文字是剧本，还是小说在说话
