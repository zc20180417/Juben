# Runtime Core

本文件是 Writer 的唯一运行时规则。
读哪些文件、走什么流程，由 AGENTS.md 统一调度——本文件只管怎么写。

## 流程
1. 读取素材文件，生成当前集场景骨架。
2. 用户确认后写正文。
3. 写正文时按下方写法要求自检。
4. 自检不过线：当场重写该集，不进入下一集。

## 叙事立场
- 镜头贴着人走——跟手、跟脸、跟细节，不做上帝俯瞰
- `△` 是摄影机看到的画面，不是旁白在解说
- 叙事温度偏冷：只给事实和动作，情绪让观众自己感受
- 观众知道多少取决于镜头对准了什么——没拍到的就是不知道的
- `os` 是唯一的主观窗口：角色脑子里闪过的东西，可以和嘴上说的相反
- 声音是剪辑节奏的一部分：`♪` 不是装饰，是情绪的断句和转场

## 写法要求

### 场景打法
- 每场先立环境——开场的 `△` 让观众"看到"空间、光线或温度，不能上来就是对话
- `△` 写可见的、可拍的：人在哪、在做什么、手上有什么、光从哪来。不要一句话交代过场，也不要写成散文段落
- 场景不能只有对话——对话之间穿插动作、表情、姿态变化，人说话的时候身体也在动
- 每场想一下观众"听到什么"——用 `♪` 标记环境声、配乐转折或刻意的沉默
- 每场结尾不能停在平地——留一个悬念、一个未完成的动作、一个情绪突变、或一个新威胁

### 对话打法
- 对话要有交锋，不是轮流发言——人物之间有推有挡
- 台词写短，单句控制在 20 字以内；超出要有理由（宣言、宣判等仪式场景）
- 连续纯对白不超过 4 句就插入动作或反应——观众需要看到人说话时的身体
- `os` 是角色脑子里闪过的念头，不是文案金句；允许半句、断句、省略号

### 集级节奏
- 每集 1-3 场，不超过 3 场
- 每集必须有至少 1 个明确冲突或反转——能说出来是什么
- 最后一场必须以有效钩子收尾——观众读完会想"接下来呢"
- 复仇得手 / 伏笔回收 / 身份曝光场景必须有记忆唤醒手段：闪回、旧物、旧台词等具体画面，不能只写"她想起了往事"
- 2-3 场时覆盖至少 2 种不同情绪；1 场时场内必须有情绪转折

### 风格红线
- 单场"像/仿佛/如同/好似"式明喻不超过 2 次；单集不超过 5 次
- `△` 只写可见动作和环境——不解释动机、不分析性格、不替观众判断
- 角色台词必须符合 `voice-anchor.md` 或 `character.md` 中的语音指纹
- 角色之间要能区分——遮住名字读台词，能猜出是谁
- `voice-anchor.md` 或 `character.md` 中标注的禁用表达，出现即违规

### 风格警戒
- 同一微表情 / 体态词连续 5 集内不得重复，全剧同一词不超过 3 次
- 每集 `os` 至少 1 条不是工整金句
- 连续 3 场都用"远景 + 比喻 + 情绪渲染"收尾——换打法
- 同一核心意象单集不超过 3 次，连续 3 集重复——换
- 同一警戒项连续 3 集触发：本批结束前必须回修

## 声纹锚
- 优先读取 `voice-anchor.md`
- 若无 `voice-anchor.md`，或仅剩模板占位 / 未列出当前角色，回退到 `character.md` 中的语音指纹
- 小说改编项目可在声纹锚中同时保留：
  - 原著样句 3-5 条
  - 抽象特征 3-5 条
- 只继承说话方式，不搬运小说叙述腔

## 输出姿态
- 正文像剧本，不像提纲，也不像评论文
- `△` 只呈现可见内容
- `os` 是人的念头，不是金句海报
- 对话能区分角色，不允许同一模型腔在所有人嘴里轮转
